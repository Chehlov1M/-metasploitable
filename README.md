## Задание 2. Режимы сканирования Nmap и анализ трафика

Трафик перехватывался с помощью Wireshark на интерфейсе `vboxnet0` (изолированная сеть VirtualBox). Для каждого типа сканирования был сохранён отдельный файл захвата в формате `.pcapng`. Все файлы размещены в папке `captures/` репозитория:
*   `syn_scan.pcapng`
*   `fin_scan.pcapng`
*   `xmas_scan.pcapng`
*   `udp_scan.pcapng`

### Сравнительная таблица режимов сканирования

| Режим | Флаги TCP | Принцип работы | Реакция сервера (Metasploitable) | Особенности в Wireshark |
| :--- | :--- | :--- | :--- | :--- |
| **SYN (`-sS`)** | `SYN` | Полуоткрытое соединение: Nmap отправляет `SYN`. Если порт открыт — получает `SYN‑ACK` и сразу шлёт `RST`, не завершая рукопожатие. Если закрыт — получает `RST`. | **Open:** `SYN → SYN‑ACK → RST`<br>**Closed:** `SYN → RST` | Чёткие пары пакетов. Видно, что Nmap намеренно разрывает соединение. Требует прав root (raw sockets). |
| **FIN (`-sF`)** | `FIN` | Согласно RFC 793, закрытые порты должны ответить `RST`, а открытые — проигнорировать пакет (не отвечать). | **Open:** Нет ответа<br>**Closed:** `FIN → RST` | В трафике много «тишины» (отсутствие ответа). Результат отображается как `open|filtered`. Работает корректно только на Unix‑системах. |
| **Xmas (`-sX`)** | `FIN + PSH + URG` | Аналогично FIN‑сканированию, но пакет содержит сразу три флага — это нестандартное поведение, заметное для защитных систем. | **Open:** Нет ответа<br>**Closed:** `Xmas → RST` | В деталях пакета явно видны три флага (`FIN`, `PSH`, `URG`). Легко детектируется системами обнаружения вторжений (IDS). |
| **UDP (`-sU`)** | N/A (UDP) | Отправляется пустой или специфичный UDP‑пакет. Закрытый порт обычно возвращает ICMP‑сообщение об ошибке. | **Open:** Обычно нет ответа (либо специфичный ответ от сервиса)<br>**Closed:** `UDP → ICMP Port Unreachable` | Преобладают ICMP‑сообщения типа «Port Unreachable». Сканирование медленное из‑за таймаутов ожидания ответа. |

### Примеры фильтров для анализа файлов захвата

Для детального изучения файлов `.pcapng` в Wireshark использовались следующие фильтры:

*   **Поиск SYN-пакетов:** `tcp.flags.syn==1 and tcp.flags.ack==0`
*   **Поиск Xmas-пакетов (все 3 флага):** `tcp.flags.fin==1 and tcp.flags.push==1 and tcp.flags.urg==1`
*   **Анализ UDP-трафика и ошибок:** `udp or icmp`
*   **Изоляция трафика целевой машины:** `ip.addr == 192.168.56.101`
